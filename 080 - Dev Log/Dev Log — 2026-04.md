- Reset mínimo: `box-sizing: border-box` em `*`, `margin/padding 0` em `html, body`, body com `font-family: system-ui, -apple-system, ...`, `font-size: 16px`, `line-height: 1.5`, `padding: var(--space-4)`, transição suave de `background` e `color`.
- Botão `#theme-toggle` flutuante: `position: fixed`, top/right `var(--space-4)`, 44×44 px (touch target da REGRA 7), `--bg-secondary` de fundo, borda `--border`, `--radius-md`, sombra `--shadow-sm`. Hover usa `--bg-tertiary` e `--shadow-md`. Focus visível usa `--accent` na borda (acessibilidade).
- Persistência em `localStorage('assistente-theme')` com valores `'light'` ou `'dark'`. Se chave ausente, default é dark.
- **Bootstrap anti-FOUC** inline no `<head>` (antes do CSS aplicar): IIFE que lê o localStorage e aplica `html.light` antes da pintura, eliminando flash de tema errado quando o usuário tem light salvo.
- Handler do botão também é IIFE — atualiza ícone (`☀️` no escuro = "vira pro claro"; `🌙` no claro = "vira pro escuro"), liga `addEventListener('click')`, sincroniza ícone na inicialização.
- Validado em produção pelo Pedro: troca de tema funciona, persiste após reload, `Hello World · Assistente` e `Banco: conectado` continuam aparecendo.

### Decisões tomadas durante a execução
- **Anti-FOUC com IIFE no `<head>` antes do `<style>`:** padrão clássico. Como a classe `light` é aplicada antes do CSS calcular as variáveis, o navegador pinta a tela já com o tema correto. Sem isso, primeira pintura sairia em dark e "piscaria" pra light depois — ruim em modo claro.
- **Ícone inicial estático = `☀️`** no HTML (combina com tema dark default). Se o tema salvo for light, `updateIcon()` corrige imediatamente no JS, antes do usuário ver. Essa milissegundo de execução de IIFE acontece antes da pintura, sem flicker visível.
- **CSS variables só no `index.html`**, não em arquivo separado. Decisão deliberada — o backlog manda criar `js/core/...` só nas Tarefas 1.2+. Mover o `<style>` pra `css/` só faz sentido quando começarmos a ter múltiplos componentes; por enquanto o HTML é uma página só.
- **Cores de marca** (`--accent: #5B6AF0`) mantidas iguais em dark e light — coerência visual com PWA antigo. Apenas backgrounds, textos e bordas mudam entre temas.
- **Accent-hover** definido (`#7280F5`) mas ainda não usado em nada — preparação pra próximas tarefas (botões primários, links). Não desperdício, é semântica do design system.

- `js/app.js` criado: entry point que importa `supabase` do módulo, define `initApp()` async que faz `from('teste').select('msg').limit(1).single()` e atualiza `#status` com `Banco: ` + msg ou `Erro: ` + message. Chama `initApp()` no fim. Sem `onAuthStateChange` ainda (Tarefa 1.5).
- `index.html`: removido o script inline antigo da Tarefa 0.2 (-21 linhas), substituído por (a) tag UMD do Supabase mantida (script clássico — define `window.supabase` antes do module rodar), (b) `<script type="module" src="js/app.js">`. CSS, design system e botão de tema intocados.
- Validado por Pedro: continua aparecendo `Hello World · Assistente`, `Banco: conectado` (vindo via novo módulo), botão de tema funciona, console do Safari sem erros.

### Decisões tomadas durante a execução
- **UMD do Supabase mantido como script clássico** (não foi pra um module via `import` do CDN). Razão: o SDK distribui um build UMD que polui `window.supabase` — usar via `window.supabase` no `js/core/supabase.js` é o padrão do projeto descrito no CLAUDE.md (REGRA 5 mostra exatamente `supabaseJs.createClient`). Importar via ESM seria um caminho diferente (ex: `import { createClient } from 'https://esm.sh/...'`) mas exigiria refatorar a convenção. Mantive o caminho descrito no contrato.
- **Ordem de carga garantida pelo navegador:** UMD é blocking (executa síncrono no parse do HTML), `type="module"` tem defer implícito (executa após DOMContentLoaded). Logo, `window.supabase` SEMPRE existe quando `js/core/supabase.js` resolve. Sem race condition.
- **`js/modules/.gitkeep`** criado pra que o git preserve a pasta vazia. Sem isso, ela sumiria no commit (git ignora diretórios vazios). Vai ser populada nas Tarefas 1.7+ (router e os módulos de domínio).
- **Refatoração autorizada:** a palavra "refactor" no commit message (`refactor: extrai cliente supabase pra js/core/supabase.js`) é coerente com REGRA 2 (proibido refatorar sem permissão) — Pedro autorizou explicitamente no prompt da tarefa.
