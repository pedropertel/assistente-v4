## 2026-07-06 — 3.E ✅ Streaming SSE token-a-token

### Análise prévia (3.E.0 — pedida pelo Pedro)

Achados que mudaram o desenho:

1. **Streamar só o texto resolvia metade da espera** — o Roteador
   come 1.2-2.2s não-streamáveis no início. Solução: evento SSE
   `router` emitido assim que a persona é decidida → front mostra
   chip + "digitando…" ~2s antes do 1º token.
2. **EventSource não faz POST** e `supabase.functions.invoke` não
   expõe stream → novo helper `invokeFunctionStream` (fetch +
   ReadableStream + parser SSE mínimo) em `core/supabase.js`.
3. **Erro pós-primeiro-byte** não tem mais status HTTP → vira
   evento `error` com o mesmo shape do JSON de erro.
4. **Bônus aprovado:** histórico buscado em PARALELO com o
   Roteador (era em série) — ~300-500ms a menos POR mensagem,
   nos dois modos.

### Entregue

| Commit | O quê |
|---|---|
| `47f6f11` (3.E.1, Edge v46) | Flag `stream: true` opt-in. Pipeline fatorado e compartilhado (JSON = comportamento v45 idêntico). Eventos `router`/`delta`/`tool`/`done`/`error`. `client.messages.stream()` + `finalMessage()` por volta do loop de tools (resto do loop inalterado). `corpoErro()` fatorado pros 2 modos. Desconexão do cliente não aborta INSERTs. |
| `abcd6fc` (3.E.2, front) | `invokeFunctionStream` em `core/supabase.js` (auth igual ao invoke: session token + apikey). `enviarMensagem` streamando: bolha criada no evento `router` com chip da persona, deltas appendam texto, `tool` mostra "⚙️ executando ação…", `done` → `carregarHistorico()` reconcilia com métricas do banco. Sem mudança de HTML/window bridge. |

### Validação

- curl modo JSON: idêntico à v45 (e `persona_escolhida: null` agora
  é null literal — fix do Roteador da 3.I confirmado).
- curl modo SSE: `router` → `delta`s → `done` com payload completo.
- Pedro no preview (3.E.3): "funcionou, ficou rápido e mostra
  parcial". Latência percebida: chip em ~1.5s vs 4-13s de tela
  parada antes.

### Rollout seguro

Produção (main) segue com front sem stream até o aprovado — a Edge
v46 serve os dois modos simultaneamente. Rollback do streaming =
remover `stream: true` do front (1 linha).
