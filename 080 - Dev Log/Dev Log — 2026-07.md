### Decisão de filosofia (Pedro, 2026-07-06)

**Tools são capacidades do SISTEMA, não da persona. Persona define
o TOM, não o PODER.** Motivo: tool presa a persona cria o bug de
UX mais enganoso possível — persona sem a tool responde "Anotado ✓"
sem gravar nada (validado 3× nos testes). Exceção mantida: tools
com credenciais/writes de risco (Meta) continuam gated por persona.
Na 3.G, registro de tools migra pra `configuracoes`.

### Fix no prompt do Roteador (UPDATE com OK do Pedro)

O seed da 3.D tinha 3 defeitos, todos validados em teste real:

1. Enum de saída `"marcos|bruno|marcela|alemao|null"` — **Marina
   não era opção válida**; nunca seria escolhida (nem com a lista
   dinâmica B14, que informa existência mas não regra de uso).
2. Sem regra de captura de ideia → "anota essa ideia sobre a
   CEDTEC" ia pro Marcos (que fingia salvar).
3. `null` entre aspas ensinava o modelo a devolver a STRING
   "null" — origem dos warnings `persona_invalida` desde a 3.D.

Correção: enum com marina, regra 0 (captura de ideia = prioridade
máxima, mesmo mencionando empresa), null literal JSON. Novo
contexto: 2.675 chars (era 9.350 — o tamanho antigo incluía
padding de espaços de um paste antigo).

### Validação fim-a-fim
